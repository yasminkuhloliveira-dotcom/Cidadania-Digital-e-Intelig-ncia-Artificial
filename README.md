# Portal Conexão Segura — Cidadania Digital e IA 🛡️

Repositório estruturado para a campanha de conscientização comunitária focada na identificação de mídias sintéticas manipulas por inteligência artificial (Deepfakes) e no combate à desinformação automatizada na sociedade contemporânea.

## 🎯 Objetivos do Projeto
* Desenvolver uma interface semântica e acessível (HTML5) para instrução cidadã.
* Implementar um ecossistema visual de alta fidelidade e responsivo (CSS3 Flexbox).
* Fornecer interatividade lógica através de um simulador dinâmico de casos de desinformação (JavaScript DOM).

## 🛠️ Tecnologias Utilizadas
* **HTML5 Semântico:** Estruturação limpa através de tags nativas de navegação e agrupamento (`<main>`, `<section>`, `<article>`, `<form>`).
* **CSS3 Moderno:** Arquitetura baseada em variáveis nativas de estilização (`:root`) para suporte a Alto Contraste e regras fluidas de posicionamento com Flexbox.
* **JavaScript (ES6+):** Captura de eventos baseada em `FormData`, controle dinâmico de exibição de elementos e persistência de dados de preferência com `localStorage`.

## 🤖 Engenharia de Prompts (Documentação Ética de IA)
Em conformidade com os critérios oficiais de originalidade, os seguintes comandos foram empregados para o refinamento da arquitetura deste projeto:

1. **Prompt para Estrutura Semântica (HTML/CSS):**
   > *"Gere a estrutura de um portal de cidadania digital voltado ao combate de deepfakes. O código precisa conter seções explícitas utilizando as tags main, section e footer, além de um formulário de interação direta. O design deve ser responsivo através de regras baseadas em Flexbox e possuir paleta de cores voltada ao universo tecnológico de segurança digital, suportando um modo escuro controlado por atributos data-theme no elemento raiz."*

2. **Prompt para o Motor de Validação (JS):**
   > *"Crie um script em JavaScript vanilla que gerencie um simulador educacional de detecção de mídias. O código deve processar um array de objetos contendo descrições de casos de desinformação, validar as escolhas enviadas através da API FormData, atualizar contadores de acertos e erros na tela sem recarregar a página e gerenciar a persistência do modo de acessibilidade no localStorage."*
